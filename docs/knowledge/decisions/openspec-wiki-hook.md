# Auto-harvest: a Stop hook closes the OpenSpec→wiki loop

## Decision

The archive→wiki harvest is enforced by an **optional Claude Code Stop hook**
([`wiki-after-openspec.py`](../../../skills/wiki-opsx-archive/scripts/wiki-after-openspec.py)),
not left to the agent remembering. The hook blocks stop when a change was
archived more recently than `docs/knowledge/log.md` was touched, and clears
itself the moment the wiki catches up. It is **silent, at zero token cost,
whenever the wiki is already current** — it only speaks when the wiki has
actually fallen behind the specs. Detection is a bare mtime comparison
(`openspec/changes/archive/` dir vs `log.md`) — no state file, no git, no
per-file scan.

## Rationale

Two owner goals from the [trending-repo directive](../interviews/2026-07-09-trending-goal.md):
auto-update the wiki after each OpenSpec interaction, and cut token costs. A
Stop hook satisfies both — it makes the write-back automatic regardless of how
the archive was triggered (slash command, free-form, or the wiki-opsx-archive
skill), and its silence-on-success design means the automation costs nothing
on the happy path and one line only when it catches a real miss. Enforcing the
*read-forward* direction (wiki→proposal) was deliberately left to the skill and
[read-first behavior](../../../behaviors/read-first.md): a PreToolUse block on
every propose would add tokens to every interaction, the opposite of the goal.

## Evidence

- [Trending-repo goal, 2026-07-09](../interviews/2026-07-09-trending-goal.md) —
  owner directive: auto-update docs after OpenSpec interactions; lower token
  costs.
- [lean-validation-runs](../constraints/lean-validation-runs.md) — the
  cheapness bar the silent-on-success design meets.

## Rejected

- **Editing the vendored `opsx/*.md` commands to append a harvest step** —
  those files are regenerated by `openspec init`/`update`, so the edit would be
  silently overwritten. A hook is decoupled from OpenSpec's own files.
- **PostToolUse hook matching the archive `mv`** — fires mid-turn but the
  matcher is brittle (the command string is generated) and it re-fires on every
  intermediate write. The Stop hook fires once and detects the outcome, not the
  action.
- **A hard block in the vendor-neutral core** — the hook is a Claude Code
  convenience shipped opt-in inside wiki-opsx-archive, keeping the portable
  path (skills + [read-first](../../../behaviors/read-first.md)/capture
  behaviors) untouched, per [no-vendor-lock-in](../constraints/no-vendor-lock-in.md).

Revisit triggers: archives and harvests start interleaving out of order (mtime
signal breaks → move to a manifest); a second agent gains an equivalent
end-of-turn hook worth documenting as a first-class install target.
