1. CONSTRAINTS AND WHY

- PocketLedger is a browser-only, single-page app.
  Reason: the entire product lives in `index.html`; there is no package manifest, server code, build config, or app framework in the project.

- Expenses are stored locally in the user’s browser under `pocketledger.expenses.v1`.
  Reason: the UI explicitly says expenses are “kept in this browser,” and the code loads/saves only through `localStorage`, allowing personal expense tracking without a backend.

- Money is EUR and represented internally as integer cents.
  Reason: display uses `Intl.NumberFormat` with `currency: "EUR"`, CSV exports an `Amount EUR` column, and expense records use `amountCents` to avoid floating-point money errors.

- Amounts must be positive and limited to two decimal places.
  Reason: the form uses `min="0.01"` and `step="0.01"`, while `parseAmountToCents` accepts only whole euros or one/two decimal digits and rejects anything below one cent.

- Expense categories are required and capped at 32 characters; notes are optional and capped at 120 characters.
  Reason: the form sets `required`/`maxlength`, and `normalizeExpense` trims stored data to the same limits so persisted data cannot exceed the UI contract.

- Dates must be `YYYY-MM-DD` date-input values.
  Reason: expenses are accepted only with a date, normalized expenses are rejected unless they match the date-input format, and monthly grouping depends on slicing the first seven characters as `YYYY-MM`.

- The “This month” total and category totals are based on the user’s current local month.
  Reason: `getCurrentMonthKey()` uses the local `Date`, and monthly totals include only expenses whose stored date begins with that month key.

- Expense lists sort newest first, with later-created records first when dates match.
  Reason: `compareExpenses` sorts descending by `date`, then descending by `createdAt`.

- Category filtering affects the rendered list and shown total, but CSV export always includes all expenses.
  Reason: the tests explicitly assert that category filtering narrows the list, and that CSV export downloads all expenses regardless of the active filter.

- Current-month category totals group by category, ignore older expenses, and sort by highest total first, then category name.
  Reason: the implementation groups only current-month records and `compareCategoryTotals` orders by descending cents, then locale category comparison.

- CSV export must quote commas, quotes, and line breaks using standard CSV escaping.
  Reason: `formatCsvField` wraps fields containing CSV-special characters and doubles embedded quotes; tests cover commas and quotes.

- The app must tolerate corrupt or unavailable browser storage.
  Reason: `loadExpenses` catches parse/storage errors and returns an empty list; `saveExpenses` returns false on failure and the UI reports that changes are visible but could not be saved.
