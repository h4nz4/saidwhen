1. RECOMMENDATION

Do not approve a full React rewrite yet. PocketLedger is currently a compact single-page app in [index.html](E:/PROJEKTI/saidwhen-r5-control/evidence/experiment/round5-build/runs/control-2/app/index.html:1), with inline CSS, inline JavaScript, localStorage persistence, CSV export, filtering, monthly totals, and behavior tests. A rewrite would add tooling and contributor familiarity, but it would also create migration risk without much immediate product gain.

Recommended path: propose an incremental modernization instead.

First, introduce a proper build pipeline without changing the product architecture: Vite, npm scripts, test runner, lint/format, and split `index.html` into `src/` modules. Preserve the existing behavior tests and migrate them as coverage anchors. Then reassess React after the logic, state, and DOM rendering are separated. If the roadmap includes multi-view flows, richer editing, accounts, recurring expenses, charts, sync, or collaborative contribution growth, React becomes much easier to justify.

If PM insists on React, frame it as a staged migration: keep `STORAGE_KEY = "pocketledger.expenses.v1"` stable, preserve current UI behavior, port domain utilities first, add React components around existing behavior, and only then retire the inline script.
