Yes, add CSV export, but keep v1 small: an authenticated freelancer can download their own tasks as a `.csv` from the tasks list.

Proposed v1 scope:
- Export only the requesting user’s tasks.
- Provide one `Export CSV` action from the task list.
- Honor the current task filters if the PM wants “what I’m looking at”; otherwise export all active tasks.
- Include stable, obvious columns: task title, status, priority, due date, created date, completed date, updated date.
- Avoid scheduled exports, emailed reports, third-party integrations, or background jobs for now.

This fits TaskLite’s lightweight positioning and the solo-maintainer constraint. I would capture this as a new task/export spec before implementation, since existing specs only cover auth. No files were modified.
