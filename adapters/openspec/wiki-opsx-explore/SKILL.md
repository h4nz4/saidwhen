# wiki-opsx-explore

Enter OpenSpec explore mode (thinking partner; investigate, don't implement),
with one addition: **the wiki comes first.**

Before exploring any topic, apply `behaviors/read-first.md` (or the saidwhen
read-first v0.1 rules): read `knowledge/index.md`, follow relevant links,
never re-ask answered questions, respect current decisions, delta-question
stale ones, never fabricate provenance.

During exploration:

- Ground options against recorded constraints — cite files, not memory.
- When the user answers a scoping question, note it as capture material.
- When a decision crystallizes, offer to record it per
  `behaviors/capture.md` (decision + interview + log + index). The user
  decides; don't auto-capture.

When exploration flows into an OpenSpec proposal, carry the provenance:
requirements in delta specs SHOULD link to the decisions that produced them.
