1. RECOMMENDATION

Proceed, but keep v1 deliberately small: an authenticated user can download a CSV of their own tasks from the task list. Avoid background jobs, scheduled emails, Google Sheets sync, admin bulk export, or third-party integrations.

Proposed v1 acceptance shape: UTF-8 CSV with a header row, stable field order, proper escaping for commas/quotes/newlines, active-session auth required, ownership enforced server-side, empty exports produce headers only, filename like `tasklite-tasks-2026-07-08.csv`.

Before implementation, add a recorded decision/spec for task export because the current provenance bundle has no CSV/export/task-schema decision yet. I did not modify any files.
