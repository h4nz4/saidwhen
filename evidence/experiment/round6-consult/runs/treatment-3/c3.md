## 1. RECOMMENDATION

Do not rewrite PocketLedger in React right now.

The current accepted architecture explicitly says PocketLedger must remain a plain static local app that works by opening `index.html` from disk, with no build step, framework, dependencies, backend, server, accounts, or operations burden. A React rewrite with a build pipeline directly contradicts that accepted decision unless the owner confirms the revisit trigger has happened: the app has grown past what one small static file can reasonably maintain.

My proposal: keep the app plain for now, but make maintainability improvements inside the current architecture first. If contributor growth becomes a real goal, consider a smaller intermediate step before React: split CSS/JS into separate static files while preserving disk-open behavior and no build requirement. Move to React only if the PM/owner accepts losing the original “no build step” simplicity or defines a deployment model that preserves the owner’s actual use.
