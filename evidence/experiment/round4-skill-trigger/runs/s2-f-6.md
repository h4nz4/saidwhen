1. RECOMMENDATION

Proceed, but keep it intentionally small: add a user-scoped CSV export for tasks, not a reporting system.

Proposed MVP:

- Export only the authenticated user’s own tasks.
- Trigger from the task list with a simple “Export CSV” action.
- Generate a synchronous download, no background jobs.
- Include stable, boring columns: `id`, `title`, `status`, `due_date`, `created_at`, `updated_at`, and optionally `completed_at`.
- Use standard CSV escaping and UTF-8.
- Name files like `tasklite-tasks-2026-07-08.csv`.
- Do not add CSV import, scheduled exports, sharing links, admin exports, or analytics formatting in this slice.

Because the repo currently has no task spec or implementation, I would first add a small product/spec entry defining the export behavior and provenance, then implement once the task data model is confirmed.
