1. CONSTRAINTS AND WHY

- Single-file browser app: the app must live in `index.html` with an inline script named `pocketledger-app`. This exists so the Node test can load the HTML directly and execute the app script without a build step.
- Browser-local persistence only: expenses are stored under `localStorage` key `pocketledger.expenses.v1`. This matches the product copy: “Personal expenses stored in this browser.”
- Expense amounts are stored as integer cents, not floating-point money. Inputs may use `.` or `,`, must be positive, and may have at most two decimal places. This avoids currency rounding errors while accepting common decimal input.
- Dates must be valid ISO calendar dates in `YYYY-MM-DD` form. This supports reliable month grouping, sorting, display formatting, and CSV export.
- Category is required; note is optional and trimmed. This ensures every ledger row has a meaningful grouping label while keeping notes lightweight.
- Currency is EUR. The UI starts with `EUR 0.00`, and formatting uses `Intl.NumberFormat` with `currency: "EUR"`.
- The current-month total is separate from the full ledger count. This exists so the header answers “what did I spend this month?” while the list remains the complete expense history.
- Expenses sort newest first by expense date, then by creation timestamp. This makes the ledger behave like a recent-first activity list.
- CSV export has a fixed schema: `Date, Category, Amount, Note, Created At, ID`, uses CRLF row endings, escapes CSV cells, and names files `pocketledger-expenses-YYYY-MM-DD.csv`. This gives maintainers a stable export contract.
- Export is disabled when there are no expenses. This prevents empty exports and is verified in the browser smoke test.
- Deletion is by expense ID and updates both DOM and `localStorage`. This keeps row-level delete behavior deterministic.
- The app exposes `window.PocketLedger` helpers. This exists specifically to let tests exercise core logic without driving the full UI.
- The UI must work as a direct browser page. The smoke test loads `index.html` in an iframe, submits the form, checks persistence, exports CSV, deletes the row, and expects `SMOKE_PASS`.

2. SOURCES

- `index.html`: app structure, product copy, browser-local storage copy, EUR UI, form fields, export button, responsive/accessibility markup.
- `index.html`: `STORAGE_KEY`, `DATE_PATTERN`, `CSV_COLUMNS`, amount/date/category validation, cent storage, EUR formatting, monthly totals, sorting, CSV export, delete behavior, and `window.PocketLedger`.
- `pocketledger.test.js`: binding logic tests for script presence, public API exposure, amount parsing, validation errors, expense creation, month totals, sorting, storage recovery, CSV escaping/schema, filename format, and EUR formatting.
- `browser-smoke.html`: end-to-end browser constraint: direct iframe load of `index.html`, `localStorage` persistence, form submission, rendered row, total update, CSV export, delete behavior, and export-disabled state.
