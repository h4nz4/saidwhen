1. RECOMMENDATION

Approve a small v1 CSV export, scoped tightly: an authenticated user can export their task list as a generated-on-demand `.csv` file. Keep it synchronous, no scheduled exports, no emailed reports, no third-party integrations.

Proposed v1 behavior: export visible/currently filtered tasks unless PM chooses “all my tasks”; include stable columns such as task title, status, priority, due date, created date, updated date, and project/list if TaskLite has that concept. Use UTF-8 CSV with proper escaping, a header row, and a predictable filename like `tasklite-tasks-YYYY-MM-DD.csv`.

This fits TaskLite’s solo-maintainer constraint: useful freelancer data portability without adding operational surface.
