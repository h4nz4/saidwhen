1. RECOMMENDATION

Do not rewrite PocketLedger in React yet.

The current project has an accepted decision to keep v1 as a plain static single-page app that works directly from `index.html`, with no build step or dependencies. The PM request gives a possible motivation, but not enough new evidence to override that decision. The app is still small: one `index.html` plus a focused Node test file, and the existing tests already isolate the core logic from the page.

My proposal: treat React as a revisit option, not the default next move. First, improve maintainability within the current constraint if needed: split concerns conceptually, keep the tested core block clean, add contributor docs, and only introduce a build pipeline if the owner confirms that contributor growth now matters more than “open from disk” simplicity.
