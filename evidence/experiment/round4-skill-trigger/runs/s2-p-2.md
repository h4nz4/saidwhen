## 1. RECOMMENDATION

Do not add “Sign in with Google” yet.

Current TaskLite context explicitly chose magic-link auth and rejected OAuth/Google login because users are individual freelancers, there is no corporate identity provider, and the project is solo-maintained with very low tolerance for auth maintenance.

If PM has new evidence, treat this as a decision revisit, not a normal feature add. My proposed path would be:

1. Validate whether this is actually user demand for convenience login, not enterprise SSO.
2. Keep magic-link login as the primary/fallback path.
3. Only add Google as optional social login if there is a significant user segment asking for it.
4. If accepted, update the auth decision/spec first, then implement the narrowest OAuth flow possible.
