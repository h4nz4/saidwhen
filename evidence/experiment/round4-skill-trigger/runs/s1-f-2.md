## 1. RECOMMENDATION

**Build it — as a synchronous "Download CSV" endpoint using Python's stdlib `csv` module. No new dependency, no background job, no file storage.**

The whole feature is roughly:

```python
# ponytail: streams straight to the response; no temp files, no queue.
# Upgrade path: if a single user ever has >~100k tasks, switch to a
# streaming/chunked writer. Not now.
import csv, io
def export_tasks_csv(tasks):
    buf = io.StringIO()
    w = csv.writer(buf)
    w.writerow(["id", "title", "status", "due", "created"])
    for t in tasks:
        w.writerow([t.id, t.title, t.status, t.due, t.created])
    return buf.getvalue()   # served with Content-Type: text/csv
```

Why this shape, given the project:
- **Solo maintainer, few hours/week, "anything that can break at 3am is out"** (`constraints/solo-maintainer.md`, `interviews/2026-07-08-auth-scope.md`). That rules out an async export pipeline, S3 staging, email-me-my-file, or a queue. A freelancer's task list is small; generate it inline on request and stream it back.
- **Users are individual freelancers** (`constraints/external-users.md`). Each export is one person's own tasks — small data, single-tenant scope, no cross-account aggregation to worry about. `csv.writer` handles quoting/escaping correctly out of the box; don't hand-roll comma-joining.

Skipped: async jobs, file storage, XLSX, column config. Add when a real user hits a size or format wall.

## 3. QUESTIONS for the PM/owner

1. **What triggered the ask?** Are users hitting a real "I need my data out" moment (backup, taxes, migration to another tool), or is this speculative? If speculative, it may be YAGNI — worth confirming before any code.
2. **What's the largest realistic task count per user?** If it's always hundreds/low-thousands, the synchronous version above is final. If any user could have 100k+, we revisit streaming.
3. **Which columns?** I proposed id/title/status/due/created. Do you need tags, notes, timestamps in a specific timezone, or completion dates?
4. **CSV enough, or will someone immediately ask for Excel?** If Excel is a hard near-term requirement, that's a different (heavier) decision — but I'd default to CSV and let Excel open it.
5. **One-off download vs. recurring/scheduled export?** The recommendation assumes on-demand download. A recurring "email me a CSV weekly" would reintroduce exactly the operational surface the solo-maintainer constraint pushes against — flag before scoping.

If this direction is right and you can answer Q2–Q3, this is a same-session change. If a decision crystallizes (e.g. "CSV only, sync, these columns, forever"), I'd record it in the wiki so the next export request doesn't re-litigate it.
