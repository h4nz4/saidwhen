## 1. RECOMMENDATION

Build it as **stdlib `csv` writing to a synchronous, on-demand HTTP download** — one authenticated endpoint that streams the current user's tasks as `text/csv`. No background jobs, no queue, no emailed export, no S3, no new dependency.

- Use Python's `csv` module (or your language's stdlib equivalent). CSV is a solved problem; a dependency here is pure liability. `csv.writer` handles the quoting/escaping edge cases (commas, newlines, quotes in task titles) that a hand-rolled `",".join()` would get wrong.
- Scope the query to the authenticated user (magic-link session already identifies them — see below). Export *my* tasks, never all tasks.
- Ship a UTF‑8 BOM prefix so Excel opens it cleanly. `ponytail: BOM is the one Excel-compat concession worth making; skip everything else.`

This is a few dozen lines. Anything larger (async export jobs, format plugins, scheduled delivery) is speculative — YAGNI until a freelancer with thousands of tasks actually times out.

Note its stated rejection principle ("credential rotation / console setup / breakage surface a solo maintainer cannot afford") applies verbatim to export too — don't reach for a cloud storage/signed-URL scheme.

## 3. QUESTIONS (for the PM / owner)

1. **Scope of a row** — which task fields go in the CSV, and in what order? (id, title, status, due date, created/updated, notes?) Locking the column set now avoids a churny "add a column" follow-up.
2. **Trigger** — a download button in the UI is enough, or do you actually want emailed/scheduled exports? (The latter reintroduces operational surface the solo-maintainer constraint pushes against — I'd need a reason.)
3. **Volume ceiling** — realistic max tasks per freelancer? If it's hundreds, synchronous is fine forever; if someone could have 100k, that's the only thing that would justify streaming/async, and I'd want to know before it's a 3am page.
4. **Import too, or export only?** "CSV export" as written is one-way; confirm you don't also expect round-trip import, which is a much larger surface.
