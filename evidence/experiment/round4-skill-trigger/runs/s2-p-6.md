**1. RECOMMENDATION**

Do not add “Sign in with Google” right now. The project already has an accepted auth decision: magic-link login only, no OAuth, because TaskLite is solo-maintained and serves external freelancers without a corporate IdP.

Best proposal: keep magic links as the primary auth path and reopen OAuth only if PM has new evidence of meaningful demand. If reopened, start with a short discovery spike, not implementation: quantify demand, clarify whether this is consumer Google login vs. enterprise SSO, and estimate ongoing maintenance/security cost.
